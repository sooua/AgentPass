Your servers' keys, kept for your coding agent.

AgentPass is an MCP server that remembers how to log into your machines, so you can tell Claude Code "restart nginx on my vps" and it just happens — without your root password living in a chat log. One process. No daemon, no database, no UI.

Security model

Honest about what it does and doesn't protect.

This keeps your secrets out of a chat transcript. It does not defend against someone already logged in as you — the master key sits next to the data, exactly like your SSH keys do.

Encrypted at rest

AES-256-GCM behind a 0600 master key with OS-level ACLs. A secret never appears in a listing, an error, or the audit log.

Every login is temporary

Login files land in a 0700 directory and are wiped when the command returns, when the TTL expires, and again at next startup in case the process was killed.

Append-only audit

One line per action — what ran, on which machine, and how it exited. Secrets are never in it.

The agent gets a shell

That is the point, and it is also the risk. Your agent's own tool-approval gate is what stops a command before it runs; the audit log is what tells you afterwards.

Two steps

Wire it up once. Then just talk.

Add the MCP server

One claude mcp add line. There is nothing to start and no token to create — the vault is created the first time you use it.

Tell your agent about a machine

"Remember my VPS: …". It is encrypted immediately and addressed by the name you gave it, however you capitalize it later.

Ask for things

"Check the disk on my vps." The agent calls run, or asks for an ssh command when you want to poke around yourself.

Get AgentPass

Two commands.

Needs Node.js 22+ and an ssh client — on Windows, the Git Bash that ships with Git.

setup
$ git clone https://github.com/sooua/AgentPass.git $ cd AgentPass && pnpm install && pnpm build $ claude mcp add agentpass -- node "$PWD/dist/index.js"
